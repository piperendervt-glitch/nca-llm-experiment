# アイデアメモ: 「自動ループでAIは賢くなる」という誤解

作成日: 2026-03-31
提唱者: Robosheep（pipe_render）

---

## 核心的な主張

> 「同じタスクを同じモデルで自動ループさせるのは基本的に悪手。
> ただし外部検証器がある・異種モデルを使う・回数を制限するという
> 条件を満たせば良手になりうる。」

---

## 世間一般の誤解

```
一般的なイメージ:
  AIが自分で考える
  → 失敗を学ぶ
  → また考える
  → どんどん賢くなる
  → AGI完成

実際に起きること:
  同じモデルが自分を評価
  → 同じ盲点で評価
  → 確信度だけが上がる
  → 多様性が失われる
  → グループシンク・Model Collapse
```

---

## なぜこの誤解が生まれるか

```
人間の学習との混同:
  人間は失敗から学べる
  → なぜなら「外部の現実」という
    絶対的な検証器が存在するから

  痛い → 火は触らない（現実が教えてくれる）

AIの自己ループ:
  「現実」という検証器がない
  → 自分の出力を自分で評価
  → 間違いを間違いと認識できない
```

---

## 歴史的に繰り返されるパターン

```
1980年代: エキスパートシステムが「知識を増やせば
          AGIに到達する」と信じられた
          → 知識の組み合わせ爆発で頓挫

2010年代: 「データを増やせばAGIに到達する」
          → スケーリング則は有効だが限界がある

2020年代: 「自己ループさせればAGIに到達する」
          → 今まさにこの誤解が蔓延中
```

---

## 今回の実験での実証

```
v1の発見:
  同じモデルを3ノードでループ
  → 99/100が同じ答えに収束
  → 「賢くなる」どころか
    「確信を持って間違える」になった

これはAGI議論への実験的な反証
```

---

## 「良手になる条件」

```
外部の正解検証器がある:
  例: 数学の答えをPythonで自動検証
      コードの実行結果でテスト

評価者と生成者が異なるモデル:
  → v5/v6の異種モデルで実証済み

反復回数に上限がある:
  → steps=2〜3が最適（v6で実証）
  → 5〜6回を超えると劣化リスクが上回る
```

## 論文への組み込み案

### Introduction / Discussionへの応用

```
「AIの自動ループでAGIに到達する」という
世間一般の誤解に対して、
本実験は小型ローカルモデルで
その誤解を実験的に反証した。

同じモデルのループ（v1）:
  → 99/100が確信を持って間違える

異種モデルの協調（v5/v6）:
  → 63%の精度（+18pp改善）

→ 「何をループさせるか」ではなく
  「どう協調させるか」が重要
```

## 対策の方向性

| 対策 | 内容 |
|------|------|
| 外部検証器の導入 | 数学・コードなど正解が一意のタスクで有効 |
| 異種モデルの使用 | 盲点を分散させる（v5/v6で実証） |
| 回数制限 | steps=2〜3が最適（v6で実証） |
| 反対案専用エージェント | 構造的に多様性を保証 |
